Changes to songs.html

Handled by: Joakim.
Instead of having only one lyric-box be visible at a time (with any already open lyric-box closing whenever you opened a new one, as per the design doc), we instead opted to let the user keep several songs open. The idea behind this was simply out of practicality; to let Crows have their specific songs open for a given performance, so the can scroll between the ones they need then and there, rather than having to open them each progressively. Another small change not specified in design: longer songs will now have their lyrics split into two columns (responsively: this only happens on screens wider than 1000px). The genuine songbook for Kråkeluren is formatted in two columns - it only made sense to do it that way.

Other changes

Changes to the navbar

Handled by: Martin
The navbar is a dynamic navbar, which shrinks when the user scrolls down a certain amount of pixels. The whole navbar, with its corresponding logo will shrink to a predetermined size. This is accomplished by using JavaScript. The responsivety of the navbar is made so that when the browser window is less than 750 pixels, the logo shrinks and the navbar-menu is then changed to one single button, which when clicked will open a dropdown menu containing the navbar. The menu buttons are all clickable, and will direct to the corresponding webpage. When on an webpage, the button for the corresponding site the user is on, will be lit up in a blue color, in order to indicate which site is currently active. The buttons will also get a faint gray background when hovered over.

Image carousel (gallery.js)

Handled by: Cathrine
I used two tutorials from w3schools as inspiration (links are in the comments of gallery.js) and created a image carousel that changes images on a timer (5 seconds apart), with the possibility of clicking "dots" to change the image as well. The comments in the file explains quite thoroughly what happens and how the carousel works. I also added a small animation when the images change, but this is with CSS.

Image transition (about_styret.js)

Handled by: Cathrine
This code is used to change the images of the board members on the about_styret.html page on hover. First I loop through the nine (number of board members) empty divs that are above the names and text, and add different background images to these. We named the different jpg-files similarly (only a different number at the end) which made this loop easier and shorter to write than to manually set the background images. Then I used the functions "onmouseover" and "onmouseout" to decide in JS what will happen when the user moves the cursor over the image and off again. I defined two functions that changes the image depending on what you do, and these have a number-argument to decide which of the jpg-files to set as background image. Again, this is easier than to write nine different hover-codes in CSS.

Order items (shop.js)

Handled by: Cathrine
This code was as mentioned not in our inital plan, but we added it to make the website even more user friendly and interactive. The code is quite long, but a big part of it is getting the elements from the shop.html-file and changing the display. This functionality allows the user to add items to a cart and "order" them, see the section about shop.html above. I created a layout where the user would have to click "add to cart" on items to then display the cart, the price and two buttons. These buttons lets the user clear the cart or go to "ordering". By choosing the last one, an input field is displayed where the user have to enter their email address before "ordering". Here I also used email validation, which I had to do using JS instead of the "required" attribute, since I have functions on the "order" button that creates a problem in HTML forms. The comments explain the code in more depth as well.

Song lyrics (songlyrics.js)

Handled by: Joakim
The functionality is described above, for the song page. For specific coding details, see the commentary in the script itself - but for a brief summary: I made a list consisting of each song as an object with its own id, the display title and display lyrics (w/ formatted line-breaks). A for-loop then outputs each song and its lyrics into their own song-box on the page, with the lyrics hidden initially. A for-loop then attaches an eventlistener to each song-box, which simply toggles those specifics lyric's display within each song-box.
